The Node Modules Problem

node_modules directory gets created when we run $ ng build. It has all the prefab code for common module files. It usually contains more than 10k files, with approximate size well over 300 MB. It becomes a headache to manage them. so what to do ?

node modules was never there to be uploaded anywhere in the first place. All we require is the src files and the dependencies. They can be easily generated by using $ npm install in the correct directory. Please ensure that the package.json file is present. This problem also arises when you are in wrong directory.
Now to deploy to a web server. There we do require node_modules.
In this case we will change our approach little. Using $ ng build --prod. This is an inbuilt bundler for angular. This will generate a dist/ directory which will contain everything required for the app. Just push this folder to the web server and you will be good to go!



How to publish to github repository

This blog deals with the most simple ways to push your app to github repo, so let's get started!
create a repository online, clone it in an empty directory on your pc using $ git clone "repo_https_url". Copy all files from your project excluding the node_modules and paste in this empty repository. use the following commands to push to remote repository-

$ git add .
$ git commit -m "commit-message"
$ git push origin master



what [(ngmodel)] requires

[(ngmodel)] is an outdated but widely used method for 2 way data binding. that is get and post methodology, as we know angular does not reload a page, it needs a mechanism to load changes in the web page.

In app.modules.ts add import { FormsModule } from '@angular/forms' ; and in the imports section add FormsModuleto be able to use it in our project.



Comparing HttpModule vs HttpClientModule

HttpModule from '@angular/http' is an outdated but widely used requests package in angular. it is now evolved into HttpClientModule from '@angular/common/http'
Way to use HttpModule npm install @angular/http also we can not use map(), instead now we have to create a pipeline for accessing this ie .subscribe().pipe(map()), something like this will work.
